배경 및 시스템 구성도

Headless Jetson 환경의 문제점과 호스트-디바이스 연결 아키텍처

문제점
  • Headless Jetson에 연결된 ZED X 카메라는 별도 디스플레이가 없음
  • 수동 SSH 세션은 불안정 — 네트워크 단절 시 진행 중인 캡처가 중단됨
  • 데이터 수집 중 발열 상태, 저장 용량, 프레임 무결성을 실시간으로 확인할 수 없음
접근 방식
  • Host Dashboard (PySide6)가 SSH를 통해 Device Capture Engine을 원격 제어
  • 엔진은 분리된 백그라운드 상태로 실행 — 케이블 분리 후에도 캡처 세션 유지
  • rsync 동기화 및 3단계 연결 추적으로 운용 안정성 확보

핵심 기능 및 기술 스택

01
프레임 무손실
저장 큐가 가득 찼을 때 캡처 루프가 블로킹됩니다. I/O 부하 시 FPS가 낮아지더라도 모든 프레임을 디스크에 저장 — 단 한 프레임도 유실하지 않습니다.
02
원클릭 배포
버튼 한 번으로 rsync 동기화 및 원격 엔진 실행까지 자동 처리.
03
원자적 메타데이터
세션 종료 시 프레임 수, 하드웨어 설정 등을 원자적으로 저장 — 100% 일관성 보장.
04
발열 모니터링
Jetson의 CPU/GPU 온도 및 저장 공간 상태를 실시간으로 추적.
05
무손실 파이프라인
RAW 스테레오 이미지를 PNG로 저장, 하드웨어 설정과 1:1 동기화. 매 쓰기마다 fsync 보장.
